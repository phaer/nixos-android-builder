14  NixOS Options Reference

Configure the builder in configuration.nix (or desktop-configuration.nix for the desktop variant). Some options have run-time equivalents — see fetch-android --help and build-android --help.

nixosAndroidBuilder.artifactStorage.artifactDir

Type: absolute path

Default: "/var/lib/artifacts"

Directory where the persistent artifact storage is mounted.

nixosAndroidBuilder.artifactStorage.contents

Type: list of string

Default: [ "*" ]

list of files (or patterns) to copy from build outputs

nixosAndroidBuilder.artifactStorage.diskLabel

Type: string

Default: "artifacts"

disk label that identifies the storage partition

nixosAndroidBuilder.artifactStorage.enable

Type: boolean

Default: false

Whether to enable Storing outputs in an unencrypted, persistent disk partition.

nixosAndroidBuilder.build.branches

Type: list of string

Default: [ "android-latest-release" ]

List of valid branches that can be selected for building. If only one branch is specified, it will be selected automatically. If more than one branch is specified, a dialog menu will be shown for 30 seconds. First entry will be the default if no other one is selected.

nixosAndroidBuilder.build.lunchTarget

Type: string

Default: "aosp_cf_x86_64_only_phone-aosp_current-eng"

Name of lunch target to build. Can be overriden at run-time by passing --lunch-target to build-android.

nixosAndroidBuilder.build.repoManifestUrl

Type: string

Default: "https://android.googlesource.com/platform/manifest"

URL of a repo manifest to fetch sources from. Can be overriden at run-time by passing --repo-manifest-url to fetch-android.

nixosAndroidBuilder.build.sourceDir

Type: absolute path

Default: "/var/lib/build/source"

Directory where repo checkout is stored. Can be overriden at run-time by passing --source-dir to fetch-android and build-android.

nixosAndroidBuilder.build.userEmail

Type: string

Default: "ci@example.com"

Email address used for repo/git operations. Can be overriden at run-time by passing --user-email to fetch-android.

nixosAndroidBuilder.build.userName

Type: string

Default: "CI User"

User name used for repo/git operations. Can be overriden at run-time by passing --user-name to fetch-android.

nixosAndroidBuilder.credentialStorage.credentialDir

Type: absolute path

Default: "/var/lib/credentials"

Directory where credentials are stored. This is the mount point of the dedicated credentials partition and will be bind-mounted to /run/credstore.encrypted/

nixosAndroidBuilder.credentialStorage.enable

Type: boolean

Default: false

Whether to enable persistent credential storage on a TPM2-bound LUKS partition.

nixosAndroidBuilder.credentialStorage.encryptionFlags

Type: list of string

Default: [ "--with-key=tpm2" "--tpm2-pcrs=7" ]

Flags to pass to systemd-creds encrypt. See man (1) systemd-creds

nixosAndroidBuilder.credentialStorage.mountPoint

Type: absolute path

Default: "/run/credstore.encrypted"

Where to mount the credential directory. /run/credstore.encrypted/ is automatically searched by systemd for encrypted credentials

nixosAndroidBuilder.debug

Type: boolean

Default: false

Whether to enable image customizations for interactive access during run-time.

nixosAndroidBuilder.fhsEnv.packages

Type: list of package

Default: [ ]

Packages needed to build Android AOSP. This is mostly copied from AOSP documentation, but could probably be reduced further, as AOSP repos ship much of it in-tree (i.e. python3, jdk, etc)

nixosAndroidBuilder.fhsEnv.pins

Type: list of package

Default: [ ]

A sorted list of packages to add first, so that they “win” if there are collisions/conflicts during creation of the FHS env. Unresolved collisions will produce a warning in the build log.

nixosAndroidBuilder.imageId

Type: string

Default: "android-builder"

The image identifier written to /image-id on the ESP. Defaults to “android-builder” and should not normally need to be changed. Kept separate from system.name so that test VMs can override system.name (which the NixOS test driver uses as the Python machine variable name) without corrupting the image-id that configure-disk-image uses to validate image type.

nixosAndroidBuilder.unattended.enable

Type: boolean

Default: false

Whether to enable unattended mode.

nixosAndroidBuilder.unattended.steps

Type: list of string

Default: [ "root:start-shell-if-yubikey-found" "select-branch" "fetch-android" "build-android" "android-sbom" "android-measure-source" "copy-android-outputs" "root:lock-var-lib-build" "root:disable-usb-guard" "root:start-shell-and-shutdown" ]

list of shell commands to run unattended

nixosAndroidBuilder.yubikeys.groupA

Type: list of string

Default: [ ]

list of u2f (i.e. yubikeys) public keys for pam, as output by pamu2fcfg

pamu2fcfg -N -i “pam://nixos-android-builder” -o “pam://nixos-android-builder” -u “user”

nixosAndroidBuilder.yubikeys.groupB

Type: list of string

Default: [ ]

list of u2f (i.e. yubikeys) public keys for pam, as output by pamu2fcfg

pamu2fcfg -N -i “pam://nixos-android-builder” -o “pam://nixos-android-builder” -u “user”

services.keylime-agent.enable

Type: boolean

Default: false

Whether to enable Keylime push model agent for TPM-based remote attestation.

services.keylime-agent.logLevel

Type: string

Default: "keylime_push_model_agent=info,keylime=info"

RUST_LOG filter string for the agent. For example "keylime_agent=debug,keylime=info".

services.keylime-agent.package

Type: package

Default: <derivation keylime-agent-0.2.9>

The keylime-agent (Rust) package to use.

services.keylime-agent.settings

Type: attribute set of (string or signed integer or boolean or list of string)

Default: { }

Freeform settings for agent.conf under the [agent] section. Keys should use snake_case INI names matching the Rust keylime-agent config. Values are merged over built-in defaults (rust-keylime v0.2.9).

Note: registrar_ip, verifier_url, and CA cert paths are configured at boot from /boot/attestation-server.json (written by configure-disk-image set-attestation-server). Build-time overrides here will be replaced at runtime if that file exists.

services.keylime.autoEnroll.enable

Type: boolean

Default: false

Whether to enable automatic enrollment of new Keylime agents with measured boot policy.

services.keylime.autoEnroll.enrollPort

Type: 16 bit unsigned integer; between 0 and 65535 (both inclusive)

Default: 8893

HTTPS port for the measured boot report endpoint.

services.keylime.autoEnroll.pollInterval

Type: signed integer

Default: 10

Seconds between polling the registrar for new agents.

services.keylime.enable

Type: boolean

Default: false

Whether to enable Keylime TPM-based remote attestation server.

services.keylime.gitServer.authPort

Type: 16 bit unsigned integer; between 0 and 65535 (both inclusive)

Default: 8895

Localhost port for the attestation auth subrequest backend. Only reachable from nginx on the same host.

services.keylime.gitServer.enable

Type: boolean

Default: false

Whether to enable attestation-gated git HTTP server.

services.keylime.gitServer.port

Type: 16 bit unsigned integer; between 0 and 65535 (both inclusive)

Default: 443

HTTPS port nginx listens on for git clone requests.

services.keylime.gitServer.repoDir

Type: absolute path

Default: "/var/lib/keylime-git/repos"

Directory containing bare git repositories served to attested agents.

services.keylime.gitServer.repos

Type: list of string

Default: [ ]

Bare git repositories to create on first boot (without the .git suffix).

services.keylime.logLevel

Type: one of "DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL"

Default: "INFO"

Log level for keylime services.

services.keylime.logLevelOverrides

Type: attribute set of (one of "DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL")

Default: { "keylime.authorization.manager" = "WARNING"; "keylime.web" = "WARNING"; }

Per-logger log level overrides. Keys are Python logger qualnames (e.g. keylime.web), values are log levels.

By default, the noisy keylime.web and keylime.authorization.manager loggers are set to WARNING to suppress routine per-request INFO messages. Set to { } to restore the previous behaviour.

services.keylime.measuredBootPolicyPath

Type: absolute path

Default: "/nix/store/13bjb1y1nr3w80vksxav2np7bbhkliq0-python3.13-keylime-measured-boot-policy-0.1.0/lib/python3.13/site-packages"

Directory containing the measured boot policy Python module. Added to the verifier’s PYTHONPATH. The module must register a policy name matching measured_boot_policy_name in verifier.conf.

services.keylime.package

Type: package

Default: <derivation keylime-7.14.1-unstable-2026-04-02>

The keylime package to use.

services.keylime.registrar.enable

Type: boolean

Default: false

Whether to enable Keylime registrar service.

services.keylime.registrar.settings

Type: attribute set of (string or signed integer or boolean or list of string or attribute set of (string or signed integer or boolean or list of string))

Default: { }

Settings for registrar.conf [registrar] section.

services.keylime.tenant.settings

Type: attribute set of (string or signed integer or boolean or list of string or attribute set of (string or signed integer or boolean or list of string))

Default: { }

Settings for tenant.conf [tenant] section.

services.keylime.tls.autoGenerate

Type: boolean

Default: true

Automatically generate a self-signed CA and mTLS certificates in /var/lib/keylime/tls/ on first activation. Existing certificates are never overwritten.

services.keylime.tls.certLifetime

Type: signed integer

Default: 365

Validity period in days for generated certificates.

services.keylime.tls.subjectAlternativeNames

Type: list of string

Default: [ ]

Additional Subject Alternative Names for the server certificate. Host IPs are auto-discovered; use this for extra DNS names or IPs.

services.keylime.verifier.enable

Type: boolean

Default: false

Whether to enable Keylime verifier service.

services.keylime.verifier.settings

Type: attribute set of (string or signed integer or boolean or list of string or attribute set of (string or signed integer or boolean or list of string))

Default: { }

Settings for verifier.conf. A nested revocations attrset maps to the [revocations] INI section.
