8  Attestation Protocol

Ports, data flow, PCR policy, and PKI reference.

8.1 Architecture

8.2 Ports & Protocols

Port Service Direction Protocol Purpose
8891 Registrar Agent → Server HTTPS (mTLS) Agent registration (EK + AK identity)
8881 Verifier Server ↔︎ Agent HTTPS (mTLS) TPM quote requests, enrollment via keylime_tenant
8893 Auto-enroll Agent → Server HTTPS (server TLS, TOFU) Agent POSTs measured boot reference state
443 Git server (nginx) Agent → Server HTTPS (mTLS) Attestation-gated git clone (optional)
8895 Git auth localhost HTTP Auth subrequest from nginx to keylime-git-auth (internal)

8.3 Data Exchanged

Port 8891 — Registrar: Agent POSTs its EK, AK, and UUID (SHA-256 of EK) on first boot. Stored in SQLite.

Port 8881 — Verifier: Accepts enrollment (keylime_tenant -c add with --mb_refstate and uki policy). Periodically sends nonces, receives TPM quotes + event logs, replays through the policy.

Port 8893 — Auto-enroll: Agent POSTs {"uuid": "...", "measured_boot_state": {...}}. Daemon matches against registrar entries and auto-enrolls.

8.4 Full Boot & Attestation Sequence

8.5 Measured Boot Reference State

Generated by measure-boot-state from the UEFI binary event log:

Field Source PCR Pins
scrtm_and_bios 0 SCRTM version + firmware blob digests
pk 7 Platform Key certificate(s)
kek 7 Key Exchange Key certificate(s)
db 7 Allowed Signature Database entries
dbx 7 Forbidden Signature Database entries
uki_digest 4 SHA-256 of the booted UKI binary

8.6 PCR Validation Policy

PCR Content Policy
0 SCRTM, firmware blobs Pinned to refstate
1 BIOS settings, boot variables Accepted
2 Boot services drivers Accepted
3 Separator Separator check
4 UKI application digest Pinned to refstate
5 GPT partition table Accepted
7 Secure Boot keys (PK, KEK, db, dbx) Pinned to refstate
9 systemd-stub event tags Accepted (tags vary per UKI; NvPCR runtime extension not in UEFI log)
11 UKI PE sections + pcrphase Accepted (runtime extensions)

Pinned = event log digest must exactly match the enrolled reference state. Accepted = events are syntactically checked but any valid digest is allowed. Separator check = validates the EV_SEPARATOR event marking the boundary between pre-OS and OS-present measurements.

PCRs 9 and 11 are excluded from event-log-vs-TPM replay comparison (they receive runtime extensions not in the UEFI log) but are included in the TPM quote.

8.7 TLS / PKI

Auto-generated on the server by keylime-tls.service. Stored in /var/lib/keylime/tls/:

File Used by Purpose
ca-cert.pem All Root CA — distributed to agents
server-cert.pem / server-key.pem Registrar, Verifier, Auto-enroll Server identity
client-cert.pem / client-key.pem Verifier, Auto-enroll, Tenant Client identity for mTLS

8.8 Agent Configuration

/boot/attestation-server.json (written to ESP at build time):

{
  "ip": "10.0.0.1",
  "port": 8891,
  "verifier_port": 8881,
  "ca_cert": "-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----\n"
}

At boot, keylime-configure-agent reads this and patches /run/keylime/agent.conf.
